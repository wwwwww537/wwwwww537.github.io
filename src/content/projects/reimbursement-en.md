## Design Philosophy
Built with a microservices mindset to solve rigid approval processes and messy attachment management in traditional corporate finance.

## Core Logic
### 1. Dynamic Workflow Engine
* **Conditional Routing**: Automatically routes claims based on amount, cost center, and applicant level.
* **Versioning**: Supports independent process versions for different categories (Travel, Office, etc.).

### 2. User Experience
* **Component-based UI5**: Provides a consistent, enterprise-grade interface.
* **Mobile First**: Seamlessly embeds into mobile apps for on-the-go approvals.

## Performance
* Leverages .NET Core async patterns to handle high-concurrency claim tracking.
